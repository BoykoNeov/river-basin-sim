# Scenarios

Example scenario configs (TOML) and saved command logs. A scenario is
**config + parameter fields + command log**, which together fully determine a run —
this is the reproducibility and sharing story (HANDOFF §7.1, §7.4).

Example configs land here as milestones add the features they exercise (uniform
rainfall at M1; spatially-varying fields, inflow hydrographs and open boundaries at
M3; structures/reservoirs at M5).
